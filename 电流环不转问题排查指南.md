## 🔴 P0: SVPWM算法不一致（最可能）

### 问题描述

工程中有两套SVPWM算法，VF模式和电流环模式用的不是同一套：

- **VF开环模式** → `SVPWM_SaddleWave()` 马鞍波法
- **电流环模式** → `SVPWM_SevenSegment()` 七段式空间矢量法

VF模式能转，说明马鞍波SVPWM和硬件是匹配的。但七段式SVPWM可能存在以下问题：

1. **电压方向相反**：马鞍波代码中有负号，七段式没有
2. **增益不一致**：七段式实际增益只有 1/√3 ≈ 0.577倍
3. **相序不匹配**：两种算法的UVW三相对应关系可能不同

### 验证方法

**快速验证（5分钟）**：把电流环的SVPWM暂时换成马鞍波

**修改文件**: `user/MiniFOC/MiniFOC.c`

**修改位置**: `MiniFOC_CurrentLoop()` 函数末尾（约第290-301行）

**修改前**：
```c
/* 10. SVPWM 七段式空间矢量合成 */
float d_u, d_v, d_w;
SVPWM_SevenSegment(Ualpha / foc.bus_voltage,
                   Ubeta / foc.bus_voltage,
                   &d_u, &d_v, &d_w);

/* 11. 转换为 CCR 值 */
uint32_t duty_a = (uint32_t)(d_u * PWM_PERIOD);
uint32_t duty_b = (uint32_t)(d_v * PWM_PERIOD);
uint32_t duty_c = (uint32_t)(d_w * PWM_PERIOD);

/* 12. 设置 PWM */
TIM1->CCR3 = duty_a;  /* U 相 → CH3 */
TIM1->CCR2 = duty_b;  /* V 相 → CH2 */
TIM1->CCR1 = duty_c;  /* W 相 → CH1 */
```

**修改后**：
```c
/* 10. SVPWM 马鞍波（临时替换，用于验证）*/
uint32_t duty_a, duty_b, duty_c;
SVPWM_SaddleWave(Ualpha, Ubeta, foc.bus_voltage, &duty_a, &duty_b, &duty_c);

/* 11. 设置 PWM */
TIM1->CCR3 = duty_a;  /* U 相 → CH3 */
TIM1->CCR2 = duty_b;  /* V 相 → CH2 */
TIM1->CCR1 = duty_c;  /* W 相 → CH1 */
```

### 判断标准

- ✅ **如果电机能转了** → 确认是七段式SVPWM的问题，需要修复七段式算法
- ❌ **如果还是不转** → 排除SVPWM算法问题，继续排查下一项

### 修复方向

如果确认是七段式的问题，可能的修复方案：
1. 检查七段式SVPWM的扇区定义和相序是否正确
2. 检查输入电压的标幺值基值是否正确（是除以Vdc还是Vdc/√3）
3. 检查是否需要取反输出

## 🟡 P1: 转子角度偏差90°

### 问题描述

如果霍尔传感器的角度和电机反电动势的角度有偏差（比如偏差90°电角度），那么：

- 你以为在控制 **Iq（转矩电流）**
- 实际上电压都加在了 **Id（励磁电流）** 上
- 结果：**产生不了转矩，电机完全不转**

这非常符合"VF能转，电流环不转"的特征——因为VF是开环的，不需要知道转子角度。

### 可能的原因

1. 霍尔传感器安装位置偏移
2. 霍尔扇区映射表 `hall_sector_map[7]` 错误
3. 电角度计算方向反了（正转时角度应该递增还是递减）
4. 电流采样相序与SVPWM相序不对应

### 验证方法

**方法1：加固定角度偏移（最简单）**

修改文件：`user/MiniFOC/MiniFOC.c`

在 `MiniFOC_CurrentLoop()` 函数中，找到电角度计算的地方（约第248行），加一个偏移：

```c
/* 电角度归一化 */
elec_angle = fmodf(elec_angle, TWO_PI_F);
if (elec_angle < 0.0f) elec_angle += TWO_PI_F;

// 临时添加：测试角度偏移
elec_angle += PI / 2.0f;  // 加90°试试
// elec_angle -= PI / 2.0f;  // 减90°试试
// elec_angle += PI;        // 加180°试试
```

分别尝试 +90°、-90°、+180°，看电机会不会转。

**方法2：堵转测试**

1. 给一个固定的 d 轴电流（Id=2A, Iq=0）
2. 用手转动电机，应该能感觉到明显的定位力
3. 如果没有定位力，说明角度或电流采样有问题

**方法3：观察Id和Iq**

用VOFA+观察Id和Iq的波形：
- 如果Id很大、Iq接近0 → 角度可能偏差了90°
- 如果Id和Iq都乱跳 → 可能是相序或方向问题

### 判断标准

- ✅ **加了某个角度偏移后电机转了** → 确认是角度偏差问题，需要校准霍尔零点
- ❌ **加了偏移还是不转** → 排除角度偏差问题，继续排查

### 修复方向

1. 校准霍尔传感器的零点偏移
2. 调整霍尔扇区映射表
3. 确认正转方向对应的角度变化方向

## 🟡 P1: 电流采样相序或方向错误

### 问题描述

如果电流采样的相序和SVPWM的相序不对应，或者电流方向反了，那么：
- Clarke变换和Park变换的结果都会出错
- 电流环无法正确调节
- 电机可能不转、震荡或反转

### 验证方法

**方法1：看三相电流波形**

用VOFA+观察 `curr_value[0]`、`curr_value[1]`、`curr_value[2]` 三个波形：
- ✅ 正常：三个正弦波，相位差120°，幅值相同
- ❌ 异常：某相没有电流、相位不对、方向反了

**方法2：单相通直流电测试**

1. 只给U相通一个小的直流电流（比如0.5A）
2. 看电流采样的哪一相有读数
3. 确认U相对应的是 `curr_value[0]` 还是其他

**方法3：验证电流方向**

1. 给U相通正电流（电流从U相流入）
2. 看 `curr_value[0]` 是正还是负
3. 如果是负的，说明电流方向反了

### 判断标准

- ✅ **三相电流正常，相位正确** → 排除电流采样相序问题
- ❌ **电流波形异常** → 修复电流采样相序或方向

### 修复方向

1. 调整电流采样的相序（交换UVW的对应关系）
2. 如果方向反了，电流值取反
3. 检查ADC通道和电机相线的对应关系

## 🟢 P2: PID参数过小导致力矩不足

### 问题描述

电流环PID参数可能太小，导致输出电压不足，电机力矩不够，克服不了静摩擦。

当前参数：
```c
#define DEFAULT_CURRENT_KP       0.5f
#define DEFAULT_CURRENT_KI       0.05f
#define DEFAULT_CURRENT_LIMIT    3.0f   // 输出限幅3V
```

母线电压24V，3V的输出限幅确实比较小。如果再加上SVPWM增益的问题，实际输出电压可能更小。

### 验证方法

**方法1：调大PID参数**

修改文件：`user/MiniFOC/MiniFOC_Config.h`

```c
// 原来的参数
// #define DEFAULT_CURRENT_KP       0.5f
// #define DEFAULT_CURRENT_KI       0.05f
// #define DEFAULT_CURRENT_LIMIT    3.0f

// 临时调大10倍，用于测试
#define DEFAULT_CURRENT_KP       5.0f
#define DEFAULT_CURRENT_KI       0.5f
#define DEFAULT_CURRENT_LIMIT    10.0f
```

**方法2：直接给固定电压测试**

在电流环里，暂时绕过PID，直接给一个固定的Uq值：

```c
// 原来的PID计算
// foc.Uq = PID_Calc(&foc.current_pid);

// 临时：直接给固定电压
foc.Ud = 0.0f;
foc.Uq = 5.0f;  // 直接给5V试试
```

### 判断标准

- ✅ **调大参数后电机能转了** → 确认是PID参数太小的问题
- ❌ **还是不转** → 排除参数问题，继续排查

### 修复方向

1. 重新整定电流环PID参数
2. 适当提高输出限幅
3. 注意不要调太大导致过流

## 🟢 P2: 电流环执行频率不匹配

### 问题描述

- **配置的周期**：`CURRENT_LOOP_T = 0.00005s` (20kHz)
- **实际频率**：TIM1 PWM频率 ≈ **10.6kHz** (170MHz / 2 / 8000)

差了近一倍，会导致：
- PID的积分速度只有预期的一半
- 动态性能变差
- 但**不会导致电机完全不转**

### 验证方法

修改 `CURRENT_LOOP_T` 为实际值：

```c
// 原来的
// #define CURRENT_LOOP_T         0.00005f   // 20kHz

// 改成实际的约10.6kHz
#define CURRENT_LOOP_T         0.000094f  // ~10.6kHz
```

### 判断标准

- 这个问题不会导致电机完全不转，只会影响性能
- 可以最后再验证和修复

## 💡 其他建议

### 调试必备观测变量

建议在VOFA+里观察这些变量：
1. `Ia, Ib, Ic` - 三相电流
2. `Ialpha, Ibeta` - Clarke变换后
3. `Id, Iq` - Park变换后
4. `Ud, Uq` - PID输出
5. `Ualpha, Ubeta` - 逆Park变换后
6. `elec_angle` - 电角度

### 安全提示

- 测试时先给小电流（0.5A~1A），确认没问题再加大
- 注意观察电机温度和电流大小
- 如果电机剧烈震动或有异响，立即断电
